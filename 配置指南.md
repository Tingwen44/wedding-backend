## 第一步：理解项目结构

### 什么是后端？

简单来说：
- **前端** = 用户看到的网页（您的婚礼邀请函网站）
- **后端** = 在服务器上运行的程序，负责接收和保存数据

### 项目文件说明

```
wedding-backend/
├── server.js              ← 后端程序（核心文件，不需要修改）
├── .env                   ← 配置文件（您需要修改这个）
├── package.json           ← 依赖配置（不需要修改）
├── README.md              ← 使用说明
├── 配置指南.md             ← 这个文件
├── frontend-integration.html  ← 前端示例（可选）
└── wedding_data.db        ← 数据库文件（自动生成）
```

**您只需要修改：`.env` 文件**

## 第二步：配置环境变量

### 什么是环境变量？

环境变量就是**配置参数**，告诉后端程序：
- 在哪个端口运行
- 用什么邮箱发送邮件
- 谁是管理员

### 如何编辑 `.env` 文件？

#### 方法 1：使用记事本（最简单）

1. **打开文件浏览器**，找到 `wedding-backend` 文件夹
2. **找到 `.env` 文件**（如果看不到，可能需要显示隐藏文件）
3. **右键点击 `.env`**，选择 **"用记事本打开"**
4. 您会看到这样的内容：

```
# 服务器配置
PORT=3001

# 邮件配置（可选）
EMAIL_SERVICE=gmail
EMAIL_USER=your-email@gmail.com
EMAIL_PASSWORD=your-app-password

# 接收邮件的地址
OWNER_EMAIL=tingwen@example.com
GUEST_EMAIL=guest@example.com

# 管理员认证令牌（用于保护 API）
ADMIN_TOKEN=your-secret-admin-token-change-this
```

### 配置步骤

#### 第 1 项：PORT（服务器端口）

```
PORT=3001
```

**说明**：这是后端运行的端口号。**不需要修改**，保持 `3001` 即可。

---

#### 第 2 项：EMAIL_SERVICE（邮件服务）

```
EMAIL_SERVICE=gmail
```

**说明**：使用 Gmail 发送邮件。**不需要修改**。

如果您想用其他邮箱（如 QQ 邮箱），改成：
```
EMAIL_SERVICE=qq
```

---

#### 第 3 项：EMAIL_USER（您的邮箱）

**修改这一行**，将 `your-email@gmail.com` 改成您的真实邮箱：

```
EMAIL_USER=tingwen@example.com
```

**例子**：
```
EMAIL_USER=tingwen.guan@gmail.com
```

---

#### 第 4 项：EMAIL_PASSWORD（邮箱密码）

**这很重要！** 不是您的邮箱登录密码，而是**应用密码**。

**为什么不能用登录密码？** 为了安全，Google 不允许第三方应用直接使用登录密码。

**如何获取 Gmail 应用密码？**

1. 访问 [Google 账户安全设置](https://myaccount.google.com/security)
2. 在左侧菜单找到 **"安全性"**
3. 向下滚动找到 **"应用密码"**（如果看不到，说明您需要先启用两步验证）
4. 如果看不到 **"应用密码"** 选项：
   - 点击 **"两步验证"**
   - 按照提示完成设置（需要验证您的手机号）
   - 返回安全设置，就会看到 **"应用密码"**
5. 在 **"应用密码"** 中：
   - 选择 **"应用"** → **"邮件"**
   - 选择 **"设备"** → **"Windows 电脑"**（或您的设备）
   - 点击 **"生成"**
6. Google 会显示一个 16 位的密码，**复制这个密码**
7. 粘贴到 `.env` 文件中：

```
EMAIL_PASSWORD=abcd efgh ijkl mnop
```

**注意**：密码中有空格，复制时要完整复制。

---

#### 第 5 项：OWNER_EMAIL（新人邮箱）

这是**您（新人）的邮箱**，用来接收宾客的回复通知。

```
OWNER_EMAIL=tingwen@example.com
```

**改成您的邮箱**：
```
OWNER_EMAIL=tingwen.guan@gmail.com
```

---

#### 第 6 项：GUEST_EMAIL（宾客邮箱）

这个可以**不填**，或者填一个通用邮箱。系统会自动使用宾客填写的邮箱。

```
GUEST_EMAIL=guest@example.com
```

**可以改成**：
```
GUEST_EMAIL=wedding@example.com
```

---

#### 第 7 项：ADMIN_TOKEN（管理员密码）

这是**保护您的数据**的密码。用来查看已收集的数据。

```
ADMIN_TOKEN=your-secret-admin-token-change-this
```

**改成一个强密码**（随意组合字母、数字、符号）：

```
ADMIN_TOKEN=MyWedding2026Secret123!
```

**记住这个密码！** 以后查看数据时需要用到。

---

## 第三步：在您的网站中集成

### 您的网站在哪里？

您的网站在：`https://tingwen44.github.io/Wedding/`

### 如何让您的网站表单连接到后端？

#### 方法 1：使用提供的 HTML 示例（最简单）

1. 打开 `frontend-integration.html` 文件
2. 在浏览器中打开这个文件，就能看到一个完整的表单
3. 这个表单已经连接到后端，可以直接使用

#### 方法 2：修改您现有的网站

如果您想在现有网站上集成，需要：

1. **找到您网站的表单代码**（通常在 HTML 文件中）
2. **在表单的 `<form>` 标签中添加 JavaScript 代码**

**具体步骤**：

1. 打开您网站的 HTML 文件（用记事本或代码编辑器）
2. 找到表单的 JavaScript 代码部分
3. 在表单提交的处理代码中，添加以下代码：

```javascript
// 配置后端 URL
const BACKEND_URL = 'http://localhost:3001'; // 本地测试
// const BACKEND_URL = 'https://your-domain.com'; // 生产环境

// 表单提交处理
document.getElementById('rsvpForm').addEventListener('submit', async (e) => {
  e.preventDefault(); // 阻止默认提交

  // 收集表单数据
  const formData = new FormData(e.target);
  const data = {
    wedding_location: formData.get('wedding_location'),
    guest_name: formData.get('guest_name'),
    guest_count: parseInt(formData.get('guest_count')),
    accompanying_guests: formData.get('accompanying_guests'),
    accommodation_dates: Array.from(formData.getAll('accommodation_dates')).join(', '),
    room_type: formData.get('room_type'),
    after_party: formData.get('after_party') === 'yes'
  };

  try {
    // 发送数据到后端
    const response = await fetch(`${BACKEND_URL}/api/rsvp/submit`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
      },
      body: JSON.stringify(data)
    });

    const result = await response.json();

    // 显示结果
    if (result.success) {
      alert(result.message); // 显示成功消息
      e.target.reset(); // 清空表单
    } else {
      alert(result.message || '提交失败');
    }
  } catch (error) {
    alert('网络错误，请检查后端服务器');
    console.error('错误:', error);
  }
});
```

## 常见问题解答

### Q1：后端无法启动，显示 "EADDRINUSE"

**问题**：端口 3001 已被占用

**解决方案**：
1. 在 `.env` 中改成其他端口，如 `PORT=3002`
2. 重新启动后端

---

### Q2：邮件无法发送

**问题**：Gmail 应用密码配置错误

**解决方案**：
1. 检查 `.env` 中的 `EMAIL_USER` 和 `EMAIL_PASSWORD` 是否正确
2. 确保已启用两步验证
3. 重新生成应用密码

---

### Q3：表单提交失败，显示 "网络错误"

**问题**：后端 URL 配置错误

**解决方案**：
1. 检查后端是否运行（访问 `http://localhost:3001/api/health`）
2. 检查 JavaScript 代码中的 `BACKEND_URL` 是否正确
3. 检查浏览器控制台是否有错误信息（按 F12 打开）

---

### Q4：无法查看数据，显示 "未授权"

**问题**：ADMIN_TOKEN 错误

**解决方案**：
1. 检查 `.env` 中的 `ADMIN_TOKEN`
2. 确保 URL 中的 token 与 `.env` 中的完全相同
3. 注意大小写和特殊字符

---

### Q5：数据库文件在哪里？

**答**：在 `wedding-backend` 文件夹中，名为 `wedding_data.db`

如果要备份数据，就复制这个文件。
