## 🚀 第一步：准备 GitHub 仓库

### 1.1 创建 GitHub 账号（如果已有，跳过）

1. 访问 https://github.com
2. 点击 **"Sign up"** 注册
3. 填写邮箱、密码、用户名
4. 完成验证

### 1.2 创建新仓库

1. 登录 GitHub
2. 点击右上角 **"+"** → **"New repository"**
3. 填写仓库信息：
   - **Repository name**: `wedding-backend`
   - **Description**: `婚礼邀请函后端服务`
   - **Public**: 选择 Public（公开）
   - **Initialize this repository with**: 勾选 "Add a README file"
4. 点击 **"Create repository"**

### 1.3 上传后端代码

#### 方法 A：使用网页界面（最简单）

1. 在新创建的仓库中，点击 **"Add file"** → **"Upload files"**
2. 选择以下文件并上传：
   - `server.js`
   - `package.json`
   - `.env`
   - `.gitignore`（如果有）

3. 在 **"Commit message"** 中输入：`Initial commit: Wedding backend`
4. 点击 **"Commit changes"**

#### 方法 B：使用 Git 命令（如果您懂命令行）

```bash
cd /home/ubuntu/wedding-backend
git init
git add .
git commit -m "Initial commit: Wedding backend"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/wedding-backend.git
git push -u origin main
```

## 🚀 第二步：部署到 Railway

### 2.1 创建 Railway 账号

1. 访问 https://railway.app
2. 点击 **"Start Project"**
3. 选择 **"Deploy from GitHub"**
4. 授权 Railway 访问您的 GitHub 账号
5. 选择 `wedding-backend` 仓库

### 2.2 配置部署

1. Railway 会自动检测 Node.js 项目
2. 点击 **"Deploy Now"**
3. 等待部署完成（通常 2-3 分钟）

### 2.3 获取部署 URL

部署完成后：

1. 点击 **"View Logs"** 查看日志
2. 在日志中找到类似的信息：
   ```
   Server running on http://0.0.0.0:3001
   ```
3. 在 Railway 仪表板中，点击您的项目
4. 在右侧找到 **"Domains"** 部分
5. 复制显示的域名，例如：`https://wedding-backend-production.up.railway.app`

### 2.4 配置环境变量

1. 在 Railway 仪表板中，点击您的项目
2. 选择 **"Variables"** 标签
3. 添加以下环境变量：

| 变量名 | 值 | 说明 |
|--------|-----|------|
| `EMAIL_USER` | 您的 Gmail 邮箱 | 用于发送邮件 |
| `EMAIL_PASSWORD` | Gmail 应用密码 | 详见下面的说明 |
| `OWNER_EMAIL` | 新人邮箱 | 接收通知的邮箱 |
| `ADMIN_TOKEN` | 自定义密码 | 例如：`my-secret-token-123` |

#### 如何获取 Gmail 应用密码？

1. 访问 https://myaccount.google.com/apppasswords
2. 选择 **"Mail"** 和 **"Windows Computer"**（或您的设备）
3. Google 会生成一个 16 位密码
4. 复制这个密码到 `EMAIL_PASSWORD` 变量

### 2.5 重新部署

添加环境变量后，Railway 会自动重新部署。等待部署完成。

## 🚀 第三步：更新您的网站

### 3.1 修改前端代码

打开 `index-integrated.html` 文件，找到第 130 行左右：

```javascript
// 本地测试时使用：
const BACKEND_URL = 'http://localhost:3001';

// 生产环境部署后，改成您的域名：
// const BACKEND_URL = 'https://your-domain.com';
```

改成：

```javascript
// 生产环境部署后，改成您的域名：
const BACKEND_URL = 'https://wedding-backend-production.up.railway.app';
```

**注意**：将 `https://wedding-backend-production.up.railway.app` 替换成您实际的 Railway 域名。

### 3.2 上传到 GitHub Pages

1. 将修改后的 `index-integrated.html` 重命名为 `index.html`
2. 上传到您的 GitHub Pages 仓库
3. 等待 GitHub Pages 自动部署（通常 1-2 分钟）

## 🔧 常见问题

### Q1：部署后表单还是显示网络错误？

**A**：检查以下几点：
1. 确认 Railway 部署成功（查看日志）
2. 确认前端代码中的 BACKEND_URL 正确
3. 等待 5 分钟后重试（DNS 可能需要时间生效）

### Q2：收不到邮件通知？

**A**：检查以下几点：
1. Gmail 应用密码是否正确
2. 是否启用了"不安全应用访问"
3. 检查 Railway 日志中是否有错误信息

### Q3：如何修改后端代码？

**A**：
1. 修改 GitHub 仓库中的 `server.js`
2. Railway 会自动检测到更改
3. 自动重新部署（通常 1-2 分钟）

### Q4：如何重新部署？

**A**：
1. 在 Railway 仪表板中，点击您的项目
2. 点击 **"Redeploy"** 按钮

### Q5：免费额度用完了怎么办？

**A**：
- Railway 每月提供 $5 免费额度
- 如果超出，可以添加信用卡升级为付费账户
- 或者使用 Render（完全免费，但有休眠限制）
